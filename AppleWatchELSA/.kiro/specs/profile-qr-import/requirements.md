# Requirements Document

## Introduction

Add a direct JSON QR profile export flow from the watch profile screen. The watch should encode the current profile as a `profile_snapshot` envelope and show a scannable QR code that lets the companion webapp load that profile.

## Requirements

### Requirement 1: Profile QR Entry Point

**User Story:** As a wearer or teammate, I want a QR button on the Profile screen, so that I can share the current profile with the webapp without typing values manually.

#### Acceptance Criteria

1. WHEN the user opens Profile THEN the system SHALL show a `Scan QR` button near the bottom of the profile form.
2. WHEN the user taps `Scan QR` THEN the system SHALL open a full-screen QR presentation.
3. WHEN the QR screen is open THEN the system SHALL provide a small back control to dismiss it.

### Requirement 2: Direct JSON Profile Payload

**User Story:** As the webapp developer, I want the QR to carry a direct JSON payload, so that the webapp can import the profile without a server-side pairing table.

#### Acceptance Criteria

1. WHEN generating profile import links THEN the system SHALL encode a `profile_snapshot` envelope using `fallrisk.v1`.
2. WHEN encoding the payload THEN the system SHALL base64url-encode the minified JSON.
3. WHEN the user has edited form values but not saved them THEN the QR SHALL represent the current form snapshot.
4. The payload SHALL include participant ID, device ID, generated timestamp, source metadata, and profile payload.

### Requirement 3: Remote QR Image Generation

**User Story:** As a watch app developer, I want the QR image generated by the web endpoint, so that the watch target does not depend on unavailable QR rendering frameworks.

#### Acceptance Criteria

1. WHEN the QR screen opens THEN the watch SHALL request `https://kirohacks.vercel.app/api/profile/qr?payload=...`.
2. WHEN the QR route succeeds THEN it SHALL return an image that the watch can display with `AsyncImage`.
3. WHEN the QR route fails THEN the watch SHALL show a readable fallback message.
4. The QR image SHALL encode `https://kirohacks.vercel.app/api/profile/import?payload=...`.

### Requirement 4: Webapp Handoff

**User Story:** As the webapp teammate, I want clear route responsibilities, so that I can implement the Vercel side without reverse engineering the watch.

#### Acceptance Criteria

1. `/api/profile/qr` SHALL read the `payload` query param and return a PNG QR image.
2. `/api/profile/import` SHALL read the same payload, decode base64url JSON, validate the schema and message type, and load or redirect to the profile import UI.
3. The watch repo README or handoff notes SHALL describe these route contracts.
